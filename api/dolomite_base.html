dolomite_base.load_object.custom_load_object_helper(meta, project, locations, memory, **kwargs)[source]#

Helper function to create application-specific variants of load_object(). This allows applications to divert to different schema locations and/or loading functions for one or multiple object classes.

Parameters:

  • meta (dict) – Metadata for this object.

  • project (Any) – Value specifying the project of interest. This is most typically a string containing a file path to a staging directory but may also be an application-specific object that works with acquire_file().

  • locations (list) – Application-specific list of names of packages to be searched for object schemas. It is expected that the schema for each object specifies the function required to create that object from its file representations.

  • memory (dict[str, Any]) – Application-specific cache of restoration functions for each object schema. This avoids redundant look-ups if the same memory is recycled across calls.

  • kwargs – Further arguments, passed to individual methods.

Return type:

Any

Some kind of object.

dolomite_base.skeleton module#

This is a skeleton file that can serve as a starting point for a Python console script. To run this script uncomment the following lines in the [options.entry_points] section in setup.cfg:

console_scripts =
     fibonacci = dolomite.skeleton:run

Then run pip install . (or pip install -e . for editable mode) which will install the command fibonacci inside your current environment.

Besides console scripts, the header (i.e. until _logger…) of this file can also be used as template for Python modules.

Note

This file can be renamed depending on your needs or safely removed if not needed.

References

  • https://setuptools.pypa.io/en/latest/userguide/entry_point.html

  • https://pip.pypa.io/en/stable/reference/pip_install

dolomite_base.write_metadata module#

dolomite_base.write_metadata.write_metadata(meta, dir, ignore_none=True)[source]#

Write metadata to disk after validating it against its schema.

meta should contain at least the $schema and path keys. $schema is used to identify the schema for validation; it should be a string or a 2-tuple containing the schema name and the name of the package in which the schema file can be found.

If the metadata describes a resource with a file representation, that file should already be present inside dir at the specified path. Also, if no md5sum key is present, the MD5 checksum of the file at path is computed and added to the metadata before saving.

Any preceding ./ in the path is automatically removed.

Parameters:

  • meta (dict[str, Any]) – Metadata for an object, typically generated by a stage_object() method.

  • dir (str) – Path to the staging directory.

  • ignore_none (bool) – Whether keys with None values should be automatically removed from meta. This is useful for sanitizing messy user-supplied inputs.

Return type:

dict[str, str]

Returns:

Dictionary that can be added as the resource property of parent metadata, if the current object is a child of another object. The metadata itself is written to a JSON file at a path derived from (or equal to) meta["path"].
